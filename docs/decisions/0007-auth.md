# ADR-0007: Authentication

- **Status:** accepted
- **Date:** 2026-06-27

## Context

The platform targets small and mid-size businesses. Customers may use Google Workspace, Microsoft 365/Entra ID, or neither. Auth must support multiple providers without requiring rewrites as the customer base grows.

## Decision

Use **Better Auth** — a TypeScript-first, self-hosted auth library with Hono and Drizzle integrations.

Supported from day one:

- Email/password
- OAuth: Google, Microsoft

Supported when needed (Better Auth plugins):

- SAML/SSO (Okta, OneLogin) for enterprise customers

Alternatives rejected:

- **Auth.js** — Vue/Hono support is second-class
- **Clerk** — hosted, user data leaves our infrastructure, vendor lock-in
- **Keycloak** — too heavy for this stage (Java, significant ops overhead)
- **Lucia** — archived by the author

## Consequences

- Auth sessions and user records are stored in PostgreSQL via Drizzle — no external auth service
- Multi-tenancy: each tenant has its own user pool; tenant resolution happens at login
- If Better Auth stalls or becomes unmaintained, migration is non-trivial as auth touches every layer — monitor project health
