# ADR-0009: CI/CD and Code Quality

- **Status:** accepted
- **Date:** 2026-06-27

## Context

All code changes must pass quality, security, and style checks before merging. The release process must be automated and based on commit history.

## Decision

**CI (on every PR):**

- `commitlint` + `husky` — enforce Conventional Commits on commit messages locally and in CI
- `oxlint` — fast linting
- `oxfmt` — formatting check
- `vue-tsc` + `tsc` — type checking
- `vitest` — unit tests
- **Codacy** — static analysis and code quality gate on PRs
- **CodeQL** — security scanning (GitHub-native, runs on PR and on schedule)

**CD (on merge to `main`):**

- Build Docker image and push to GHCR
- Run `kamal deploy`
- **Semantic Release** — reads Conventional Commits, bumps version, tags release, generates changelog automatically

**Git strategy (enforced via GitHub branch protection on `main`):**

- Signed commits required
- Rebase only — no merge commits
- Linear history required
- PR required before merging — no direct pushes to `main`

## Consequences

- Semantic Release version bumps are fully automatic — no manual tagging
- `feat` commits bump minor, `fix` commits bump patch, `feat!` or `BREAKING CHANGE` bumps major
- All developers must configure GPG signing locally
- Squash merges are disabled — commit history on `main` reflects the full PR history
